= Musik in Schaumburg
:toc: left
:icons: font
:source-highlighter: highlight.js

Landing Page für Musikensembles im Landkreis Schaumburg, Niedersachsen, Deutschland.

== Überblick

Diese Seite bietet eine Übersicht aller Musikensembles, Chöre und Blasorchester im Landkreis Schaumburg.

*Betreuer:* Benjamin Marwell — link:https://bmarwell.de[bmarwell.de]

== Schnellstart

Abhängigkeiten installieren und die Seite bauen:

[source,bash]
----
bun install
bun run build
----

Vorschau der gebauten Seite im Browser starten:

[source,bash]
----
bun run preview
----

Die gebaute Seite liegt im `dist/`-Verzeichnis.

== Struktur

[source]
----
musik-in-schaumburg.de/
├── ensembles/                  # Ensemble-Daten
│   └── <slug>/

├── src/main/
│   ├── html/
│   │   ├── index.html          # Startseite (Mustache-Template)
│   │   ├── ensemble.html       # Ensemble-Detailseite (Mustache-Template)

└── package.json
----

== Ensemble hinzufügen

Um ein neues Ensemble hinzuzufügen, legt man einen neuen Ordner unter `ensembles/<slug>/` an mit einer `index.yaml`-Datei.

=== YAML-Schema

[source,yaml]
----
# ── Pflichtfelder ─────────────────────────────────────────────────────────────
title: "Name des Ensembles"
type: brass-band            # Typ-Schlüssel – siehe Tabelle unten
slug: mein-ensemble         # URL-Pfad, z.B. /ensemble/mein-ensemble/
description: >
  Kurzbeschreibung des Ensembles auf Deutsch. Kein HTML.
  Anführungszeichen immer als »Guillemets«.

# ── Allgemeine Angaben ────────────────────────────────────────────────────────
founded: 1975               # optional: Gründungsjahr (ganze Zahl)
active: true                # optional: false = aufgelöst / inaktiv (Standard: true)
member_count: 40            # optional: Ungefähre Mitgliederzahl

conductors:                 # optional: Dirigent(en) / Leitung
  - name: "Max Mustermann"
    role: "Dirigent"        # optional: Dirigent, Jugenddirigent, Chorleiterin, …

membership_fee: "60 €/Jahr" # optional: Mitgliedsbeitrag (Freitext)

# ── Bilder ─────────────────────────────────────────────────────────────────
logo:
  local: "logo.png"         # BEVORZUGT: Datei in ensembles/<slug>/logo.png
  # url: https://...        # Nur als Fallback, wenn kein lokales Logo möglich

image:
  local: "photo.jpg"        # BEVORZUGT: Datei in ensembles/<slug>/photo.jpg
  # url: https://...        # Nur als Fallback, wenn kein lokales Bild möglich

# ── Standort ──────────────────────────────────────────────────────────────────
# location: Nur setzen, wenn kein address-Block vorhanden ist ODER wenn der
# Anzeigeort vom address.city abweichen soll (z.B. "Steinbergen / Rinteln"
# statt "Rinteln"). Ist address.city gesetzt, wird es automatisch verwendet –
# location wäre dann redundant.
location: "Stadthagen"      # optional

address:
  name: "Probenraum XY"     # optional: Gebäude-/Raumname (kein Stadtname!)
  street: "Musterstraße 1"  # optional
  postcode: "31655"
  city: "Stadthagen"
  maps: "https://maps.app.goo.gl/…"  # optional: Google Maps-Kurzlink

geo:                        # optional: Koordinaten für die Kartenansicht
  lat: 52.3250              # Breitengrad (Dezimalgrad, Zahl)
  lng: 9.1360               # Längengrad (Dezimalgrad, Zahl)

# ── Proben ────────────────────────────────────────────────────────────────────
rehearsal:
  day: "Donnerstag"         # Wochentag auf Deutsch
  time: "19:30"             # optional: Uhrzeit (HH:MM)
  # location: Nur setzen, wenn der Probenort von address.name abweicht

# ── Kontakt & Web ─────────────────────────────────────────────────────────────
website: "https://example.com"  # optional

contact:                    # optional: nur öffentlich zugängliche Daten
  email: "info@example.de"
  phone: "+49 5722 …"       # optional: nur wenn öffentlich gelistet

social:                     # optional: alle Unterfelder optional
  facebook: "https://www.facebook.com/..."
  instagram: "https://www.instagram.com/..."
  youtube: "https://www.youtube.com/..."
  twitter: "https://twitter.com/..."

# ── Stichwörter ───────────────────────────────────────────────────────────────
# Erlaubte Werte stehen in src/main/keywords.yml. Unbekannte Tags erzeugen
# beim Build eine Warnung, werden aber dennoch übernommen.
# Alias: keywords (wird intern zu tags zusammengeführt)
tags:
  - Jugend
  - Kirche
  - Blasmusik
----

=== Bilder und Logos

Bilder und Logos sollten **immer lokal** im Verzeichnis `ensembles/<slug>/` abgelegt und in der YAML-Datei mit `image.local` bzw. `logo.local` referenziert werden.
`image.url` / `logo.url` dienen nur als Fallback, wenn kein lokales Asset verfügbar ist, und erzeugen beim Build eine Warnung.

== Build-Pipeline

Die Build-Pipeline (`scripts/build-pipeline.mjs`) führt folgende Schritte durch:

1. `dist/`-Verzeichnis leeren
2. YAML-Dateien der Ensembles einlesen
3. Bilder und Logos herunterladen (falls URLs angegeben)
4. Responsive WebP-Varianten erstellen (400 px, 800 px, 1200 px)
5. HTML-Seiten aus Templates generieren (Startseite + je eine Seite pro Ensemble)
6. `sitemap.xml` generieren
7. CSS, JavaScript, `robots.txt` und `.htaccess` kopieren
8. Alle Textdateien vorab komprimieren: Brotli (`.br`), gzip (`.gz`) und zstd (`.zst`)

=== Vorab-Komprimierung

Jede Textdatei in `dist/` (HTML, CSS, JS, XML, TXT, SVG) wird in drei komprimierte
Varianten erstellt. Apache liefert die passende Variante anhand des vom Browser gesendeten
`Accept-Encoding`-Headers aus (Priorität: zstd > brotli > gzip). Falls keine vorab
komprimierte Datei vorhanden ist, greift Apache auf die Echtzeit-Komprimierung via
`mod_brotli` / `mod_deflate` zurück.
